这是我建好博客之后的第一篇文章，就记录下我搭建个人博客的过程和想法吧🥸

写在开头

这篇文章并不是手把手教怎么搭建博客，只是记录一下我搭建时使用的方法，还有过程中遇到的一些问题和解决方式，然后还有一些可能会踩坑的地方。搭建个人博客的方式有很多，而且这篇文章里提到的一些坑也一定会有其他的更好的解决方法，仅作参考😉～

本博客使用Hexo搭建，如果不了解Hexo是啥，可以看看这个：Hexo官方文档

Heroku: 是真方便

我之前并没有搭建网站的经验，所以第一步便是找教程，一开始的操作参考是杰森大佬 的这篇文章：# 快速搭建个人博客——保姆级教程 ，讲得已经非常详细了，但就在我部署到Github pages的时候，始终无法显示配置好的页面，一直都只显示仓库自带的readme文档，我无论怎么刷新或者反复地用 hexo deploy 都没有用（其实这里不能只用 hexo deploy，应该要用的是 hexo clean && hexo generate && hexo deploy 才能覆盖部署，但我当时不知道🤣），折腾了好久，最后放弃了用GitHub pages建站的方法，开始寻找其他方式。

后来我在 Hexo 官方文档 里找到了一个叫 Heroku 的部署方式，非常的简单，差不多就下面几步：

  • 先在 Heroku 官网 注册账号然后创建一个app

  • 在blog根目录下先安装hexo-deployer-heroku插件

  • 修改站点配置文件 _config.yml

  • 最后在blog根目录下

就可以啦～（具体操作也可以参考这篇：Hexo-Heroku部署教学 ，说实在话，确确实实比GitHub pages 简单很多）

但为啥后来我还是放弃了Heroku，又回到了GitHub pages呢？（说白了还是我太菜）我记得最后一次出现的报错大概的意思是说，覆盖失败了，有配置错误，要我检查下配置文件…额，这你又不告诉我具体一点是因为啥我怎么查嘛？我试着Google了一下，但返回的结果几乎都是依赖GitHub pages的，这就给我整不会了呀？？然后那天我漫无目的地改了一下午但必然地没有什么卵用…

然后我还是决定回到 GitHub pages😤，然后就有了下文👇～

GitHub Pages: 还得是你

兜兜转转，最后还是回到了GitHub pages，我也很顺利地在本地搭建好了静态网站（主题啥的我都在本地先配置好了），然后我建好仓库，用 hexo clean && hexo generate && hexo deploy 部署到 Github Pages ，我兴冲冲地刷新博客页面：

一点想法或Tips

其实现在个人博客的搭建方式还是蛮多的，平台也不少，网上教程也是一找一大堆，我也差不多就是跟着教程一步一步来的，说有什么自己的心得好像还不至于😂，倒是有几个小tips可以说一说：

  • 从本地部署到托管平台的时候，要先 hexo clean 再 hexo generate && hexo deploy ，不要像我那样傻fufu地只执行一个 hexo deploy ;
  • 如果是在 GitHub pages上面部署的话，可能在提交部署后需要等待一段时间，就是等那个 pages build and deployment ，在 GitHub actions 里面可以看到，等这个过程运行完之后再去看下自己的博客有没有刷新;
  • 关于图床嘛，目前来说我感觉比较稳的是SMMS图床（我自己现在用的就是这个），或者七牛云、Imgur也还不错，然后上传到博客的话我用的是Picgo这个软件，详细用法可以看看它的官方文档;
  • 在配置主题的时候，有的地方可能会更换一些图片，在部署之前一定要看清楚原本主题模版里面对应图片的尺寸！！！ 就是像下图这样的，务必要预先把自己的图片调整成和模板图一样的尺寸，这样在网页上的显示才是正常的，不然你替换的图片会以“该多大就多大”的姿态躺在你的页面上🤪～～
